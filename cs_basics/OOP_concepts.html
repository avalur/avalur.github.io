What are objects?

A data abstraction that captures..

  1. an internal representation through data attributes
  2. an interface for interacting with an object through methods (aka procedures/functions). Defines behavior but hides implementation

ArrayList example

ArrayList is a resizable array implementation in Java's Collection framework. Unlike traditional arrays, it can dynamically grow and shrink as you add or remove elements.

How to use ArrayList

  • Declaration: ArrayList<Type> list = new ArrayList<>();
  • Adding items: list.add(item);
  • Accessing items: Type item = list.get(index);
  • Size: int size = list.size();

ArrayLists are part of the java.util package, so don't forget to import it at the beginning of your Java file.

What are attributes?

  • Data and procedures that "belong" to the class
  • Data attributes
    • Think of data as other objects that make up the class
    • For example, a coordinate is made up of two numbers
  • Methods (procedural attributes)
    • Think of methods as functions that only work with this class
    • How to interact with the object
    • For example, you can define a distance between two coordinate objects, but there is no meaning to a distance between two list objects
										  
public class Coordinate {
    // Attributes for the x and y coordinates
    private double x;
    private double y;

    // Default constructor
    public Coordinate() {
        // Initialize with default values
        this.x = 0.0;
        this.y = 0.0;
    }

    // Parameterized constructor
    public Coordinate(double x, double y) {
        // Set the x and y coordinates
        this.x = x;
        this.y = y;
    }

    // Also you need getters and setters for x and y
}

In Java, you explicitly only pass arguments to the constructor, and this is reference to this current instance of a class. In Python you have to use first argument self for it.

Define a method for the coordinate class

										  
            // Java class definition
            public class Coordinate {
                private double x;
                private double y;

                public Coordinate(double x, double y) {
                    this.x = x;
                    this.y = y;
                }

                public double distance(Coordinate other) {
                    double xDiffSq = Math.pow(this.x - other.x, 2);
                    double yDiffSq = Math.pow(this.y - other.y, 2);
                    return Math.sqrt(xDiffSq + yDiffSq);
                }
            }

Other than this (equivalent to self in Python) and dot notation, methods behave just like functions (take parameters, do operations, return).

Class Hierarchies in Java



  • parent class (superclass)
  • child class (subclass)
    • inherits all data and behaviors of parent class
    • add more info, add more behavior
    • override behavior

Which method to use?

  • A subclass can have methods with the same name as superclass
  • For an instance of a class, look for a method name in current class definition
  • If not found, look for method name up the hierarchy (in parent, then grandparent, and so on)
  • Use the first method up the hierarchy that you find with that method name

The Diamond Problem in OOP

The Diamond Problem refers to an ambiguity that arises when two classes B and C inherit from A, and class D inherits from both B and C. If there is a method in A that B and C have overridden, and D does not override it, which version of the method does D inherit: that of B, or that of C?

Note: In Java, the Diamond Problem is avoided because Java does not support multiple inheritance for classes. However, it can occur with interfaces, and is resolved using explicit method invocation and interface default methods.
